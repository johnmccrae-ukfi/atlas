# Lessons Learned

## Stage 1 — Initiation & Vision

### What worked well

- Starting with vision and governance before implementation created clarity.
- The PID provided a strong foundation for future decisions.
- ADRs helped capture key architectural decisions early.
- Mermaid diagrams worked well for GitHub-native architecture documentation.
- Conventional Commits created a clean, professional project history.
- GitHub Releases provided a useful milestone marker.
- AI-assisted planning accelerated documentation and architecture work.

### Key decisions

- Microsoft Fabric selected as the core platform.
- Medallion Architecture selected as the standard data pattern.
- AI-assisted development adopted as an engineering methodology.
- GitHub-first engineering adopted as the project delivery model.

### Improvements for future stages

- Define MVP data requirements before implementation.
- Keep scope tightly controlled.
- Continue documenting decisions as ADRs.
- Maintain the clean Git history established in Stage 1.
